NAME

require_magic.pl is intended as a ClearCase Trigger script.

Script: require_magic.pl

SYNOPSIS

If a client attempts to create a new element in ClearCase this trigger will require the environment variable MAGIC_PATH to exist and the location must contain at least one readable .magic file. ``cleartool man cc.magic'' for more information about ClearCase Magic file.

DESCRIPTION

The trigger is useable if you have customized element types, and want to have new elements typed correctly (to you custom element types) when they are added to source control.

When adding a new element types to a vob, the magic file must be changed if the file-typing should work. The default magic file exists on each client, and it is error-prone to try update or modify the magic file in each client. You can how-ever make every client see the same magic file by having each client create the MAGIC_PATH environment variable. Search Rational ClearCase support site for more information about the MAGIC_PATH environment variable.
